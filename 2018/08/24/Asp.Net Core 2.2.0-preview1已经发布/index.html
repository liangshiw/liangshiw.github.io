Asp.Net Core2.2.0-preview1已经发布

  • 梁士伟
  • 2018-08-24
  • Aspnetcore
  • 次

原文地址 ASP.NET Core 2.2.0-preview1 now available

今天我们很高兴地宣布,现在可以试用ASP.NET Core和.NET Core的下一个次要版本的第一个预览。在过去的几个月里，我们和社区里的许多人一起为这个版本进行开发，现在它已经准备好让更广泛的受众尝试它并提供反馈，这些反馈可以帮助我们更好的完善这个版本。

在哪里可以下载到2.2.0-preview1?

您可以从 在这里下载.NET Core 2.2.0-preview1版本的SDK（包括ASP.NET 2.2.0-preview1）

Visual Studio版本要求

在创建.NET Core 2.2和ASP.NET Core 2.2项目时,除了安装上面提供的sdk,使用Visual Studio的开发人员还需要安装预览版的Visual Studio 2017 (在写本文时Visual Studio预览版是15.9 Preview 1)

Azure应用服务要求

如果您使用Azure App Service运行应用程序,则应阅读这些文档安装用于托管2.2.0-preview1应用程序的站点扩展

对机器环境的影响

需要注意,这是一个预览版本,可能存在已知和尚未发现的错误。虽然.NET Core SDK和运行时是多版本共存在的，但默认SDK将成为最新的SDK。如果在安装预览SDK后遇到使用早期版本的.NET Core项目出现问题，可以根据此文档描述的global.json文件强制特定项目使用指定的sdk版本来解决,如果您遇到sdk版本有向后兼容问题,请在这里提出一个issue

2.2.0中的新功能

我们会发布一系列的博客介绍2.2.0中新加入的功能，它们会在未来几天内发布，导航链接将在本文中更新 (注:翻译将在未来几天内更新)

  • API控制器约定
  • 端点路由
  • 健康检查
  • Kestrel中的http/2
  • 改进IIS托管
  • SignalR Java客户端

除了这些功能，我们还有

  • 将web模板更新为bootstrap4和angular6

您可以在Asp.Net社区看到这些新功能的实际应用

2.2.0-Preview1的相关所有功能、错误列表和已知问题的列表请参阅我们的发行说明

将ASP.NET Core 2.1项目迁移到2.2

将ASP.NET Core项目从2.1.x迁移到2.2.0-preview1的方式很简单，打开项目的.csproj文件并将\元素的值更改为netcoreapp2.2即可,如果您的项目是.NET Framework 4.x，则无需执行此操作(因为它需要更复杂的迁移方式)。
